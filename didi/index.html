Intelligence

Didi learns.
And never forgets.

Two systems work together to make every conversation smarter than the last.

🧠

Memory System

Persistent long-term storage for personal facts — names, CPFs, addresses, preferences, account numbers. Stored in memories.md and injected automatically into every session. Didi never asks twice.

🎓

Lessons Learned

Every correction, preference or self-fix is captured automatically — silently, without asking. Stored in lessons.md and injected on every query. The same mistake is never made twice.

⚙️

Self-updating Codebase

Lessons are periodically triaged by an automated task. Each lesson is promoted to the right place: universal rules go to CLAUDE.md, skill-specific fixes go to the corresponding SKILL.md, and personal preferences go to memories.md. Didi then commits the changes to its own git repo — autonomously.

user correction → lessons.md triage → CLAUDE.md / SKILL.md / memories.md git commit + push active on next query

Skills
39 skills
Skills System
Every capability is a SKILL.md file with YAML front matter, description, and allowed tools. Claude automatically routes tasks to the right skill based on the user's intent — no explicit commands needed.
Persistent Context
Didi runs as a streaming session (max_turns=15). Heavy tasks are delegated to isolated claude -p subprocesses, keeping the main context clean. Memory and lessons persist across all sessions.
Highlight Feature

TDAH Focus Manager

Designed for ADHD brains. One rule: do three things a day.

🎯

Rule of 3

Every morning, define exactly 3 tasks: 1 must-do (non-negotiable) + 2 important. Nothing else counts until these are done.

📥

Task Deposit

New ideas and tasks go into a "deposit" during the day, so they don't interrupt current focus. Review at day close.

🌙

Day Close

End-of-day ritual: review what was done, move uncompleted tasks, clear deposit. Clean slate for tomorrow.

Highlight Feature

Scheduler

Didi can schedule tasks for itself — with full context preserved in a mailbox.

📬

Mailbox Context

When scheduling a future task, Didi writes a full prompt + context to a mailbox file. When the cron fires, the full context is injected — no information lost.

🔁

Recurring Tasks

Daily summaries, weekly reviews, monthly payment reminders — all configured as cron-based recurring schedules.

🔄

Self-Retry

If a task fails (API timeout, rate limit), Didi can re-schedule itself for a retry in 5 minutes — fully autonomous error recovery.

# Didi schedules a follow-up uv run python scheduler_manager.py add-once \ --title "Check build status" \ --prompt "Verify if deploy succeeded. If not, notify user and retry." \ --at "2026-03-10T15:30:00" \ --target inject \ --context "Deploy started at 15:00 Repo: acme/backend PR #42" \ --model claude-sonnet-4-6 # Mailbox file: tasks/42.md # Cron fires → reads mailbox # → executes with full context
